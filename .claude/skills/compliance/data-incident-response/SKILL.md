# Data Incident Response

## Inputs
- Incident description
- Affected data
- Affected subjects

## Workflow
1. Triage scope and severity
2. Build containment plan
3. Draft regulator notification (if required)
4. Draft data-subject notification (if required)
5. Hand to hard-gate-keeper

## Output
Incident response package.

Every client-facing document follows `docs/DOCUMENT_STYLE_GUIDE.md`.
No emoji. No decorative colour. Black on white. Cite statutes and
cases precisely.
